Data Import
The GOES Instrument can read a file already on the PC, using importGOESLocal

As an example, we have provided a test file in dist/workspace.
On the Commands tab, select the Importer Module, and the importGOESLocal Command.
Just copy this name from here and paste into the Filename parameter box: workspace/samples/goes/20120119_Gp_xr_5m.txt
This file has some negative values, to check they are handled correctly.
Click the 'Execute' button, and you will see the status briefly change to 'Busy'.
When the instrument again says 'Ready', you may view the data on the Chart tab.
You can display any GOES files you have previously downloaded by providing a path to the file, based on 'dist' as the working directory.

You can also retrieve a file from the remote NOAA server, using importGOESRemote

The SWPC X-ray data list naming convention was recently altered to better
support the next and future GOES satellite designation changes. SWPC
designates a Primary and a Secondary GOES Satellite for each instrument.
http://www.swpc.noaa.gov/Data/goes.html

The old data list naming convention included the designated satellite number,
e.g. G10xr_1m.txt. This required customer changes whenever the satellite
designation changed. The new Data List Naming Convention replaces the
satellite number with a 'p' for Primary and 's' for Secondary. The satellite
number is available in the Data List file header.

NOTE: SWPC Store historical data for approximately two years only.
Older data going back as far as 1996 can be obtain direct from http://www.swpc.noaa.gov/ftpmenu/warehouse.html

The new convention began Dec 1, when GOES 14 replaced GOES 10 as the SWPC
Primary GOES X-ray Satellite.
There is no Secondary GOES X-ray Satellite at this time.

Naming Convention:
2-hour lists: Gp_xr_1m.txt and Gs_xr_1m.txt
Gp_xr_5m.txt and Gs_xr_5m.txt
Full day lists: 20091201_Gp_xr_1m.txt and 20091201_Gs_xr_1m.txt
20091201_Gp_xr_5m.txt and 20091201_Gs_xr_5m.txt

On the Commands tab, select the Importer Module, and the importGOESRemote Command.
Enter *only* the file name into the Filename parameter box with the following rule:
* If the name starts with the date pattern '20YYMMDD' then use the name literally.
* e.g. 20091201_Gp_xr_1m.txt

* Otherwise, the software will prepend today's date using that pattern.
* So you only enter e.g. Gp_xr_5m.txt
* This will always retrieve the same dataset for the current day (in this case the 5min samples).
* The above is to simplify the download of a file every day, without having to explicitly enter the date.
* Note that this will not work after 2099!

PLEASE NOTE! If the download fails because of 'file not found',
CHECK that there is a data file from the satellite you specified by browsing ftp://ftp.swpc.noaa.gov/pub/lists/xray
Quite often the satellites go off-line and no files are produced for a while.
