Port Monitor Tab

The log shows an index number and the Date and Time of the Port event
The Port originating the event (e.g. Staribus, SpectraCyber)
The Stream is either Tx (transmit, sent from the PC) or Rx (receive, arrives at the PC)
Traffic indicates the bytes sent over the channel, with control characters expanded
The Hex column shows the stream traffic as a series of Hex bytes
StarScript is the scripting language statement executed by the command, or the name of the response variable
The Response Value is the data returned by the remote device
The Units refer to the response value (e.g. mV, Celsius)
DataType refers to the response value (e.g. DecimalByte, Float, Date)
The Response Status is the 16 bit field of status information. All zero means 'success'.
The Status Codes show the text mnemonic corresponding to the bits in the status field

Memory Usage Tab

The Memory Usage chart show the amount of memory used by Starbase as a function of time,
while the Observatory Monitor is running.
The memory allocated to Starbase is set on the java command line in the batch file/shell script
and is set by default to 512Mb.
You may run the Java 'Garbage Collector' at any time, by clicking the wastebin next to the
memory usage status indicator. This will force all unused memory to be released.
